elemento article:

Elemento "aside"

Es poco importante con respecto a los demás elementos como main, article, nav, section.
La idea es usarlo para las cosas que podríamos quitar de la interfaz sin problemas. Ej: publicidades, opiniones. Muchas veces, en la visualización de la interfaz, presentan una cruz para cerrar esa partecita de informacion. Aun así, sí presentan valor semántico. Este último dato lo diferencia del elemento div.

Por lo general, se usa para poner información relacionada con el contenido principal pero que no es esencial. Por ejemplo, un comentario o una cita relacionada con el tema principal.

elemento div:

Este sí que no tiene valor semántico y normalmente se usa solo para "maquetar". div viene de division. Es un contenedor genérico que se usa para agrupar contenido en bloques. No tiene significado semántico por sí mismo. Solo sirve para ordenar, organizar contenido y estructurar cosas.

CSS: Cascading Style Sheet

Es un omplemento de html que sirve para cambiar el aspecto (display/layout) de los elementos que estructuramos previamente.

Existen 3 maneras de implementarlo: interna, externa y en línea.

en línea

Trabajar dentro del elemento a traves de un atributo llamado: style. ej: fijate como estoy cambiando los atributos de esta etiqueta.
No es la manera recomendada ya que da paja tener que ir siempre al código y encontrar la etiqueta específica donde están esos datos.
De todas maneras igual se usan, y el ejemplo que dio la chica es en "newsletters" pero nidea.

Externa: es la manera recomendada

Para utilizarla, hay que usar el elemento "link" y todo esto hay que hacerlo en el head porque sino no funciona!

esto es así: para conectar esta cosa al resto de interfaces y que estas se vean afectadas por los cambios que se realicen acá, hay que vincularlas con ese link q ya viste. entoncas, la estructura para agregar cambios es la siguiente: selector1 (propiedad:valor; propiedad2:valor; etc) selector2 (lo mismo que arriba. la idea es que estas propiedades formen parte de la misma "regla" osea el selector) selector3 y así sucesivamente

Interna

Se usa en el encabezado. la diferencia con en línea es que para hacerlo inline es a través del ATRIBUTO STYLE mientras que interna es a traves del BLOQUE STYLE. El ejemplo está puesto en el encabezado de este código!

Propiedad Border:

Se puede agregar un borde a cualquier elemento. Valores posibles: none (sin borde), solid (borde sólido), dashed (borde discontinuo), dotted (borde punteado), double (borde doble), groove (borde con relieve), ridge (borde con relieve invertido), inset (borde interior) y outset (borde exterior). También se puede especificar el grosor del borde (px o %), el color y el estilo. Para implementarlo, primero hay que definir el estilo. osea: la propiedad "border-style", porque si no hacemos esto, no se van a ver los demás cambios que hagamos. Luego, se puede especificar el grosor del borde con la propiedad "border-width" y el color con la propiedad "border-color". También se puede usar la propiedad "border" para definir el estilo, el grosor y el color en una sola línea.

    Formmularios

    Los formularios son elementos que permiten al usuario enviar información a un servidor. Se utilizan para recopilar datos, como nombres, direcciones de correo electrónico, contraseñas, etc.

    El elemento principal es: "form". Los atributos más importantes son: "action" y "method"

    Atributo: method

    Sus valores pueden ser:

    • get: Ideal para búsquedas. La info queda guardada como apéndice del url. Se considera no seguro. Además tiene límite de caracteres a ingresar.
    • post: Ideal para formularios de contacto o login. Es seguro, permite adjuntar archivos.

    Atributo: Action

    Es la URL a la que se enviará la información del formulario. Si no se especifica, se enviará a la misma página.

    Atributo: type

    Especifica el tipo de campo que se va a utilizar. Los valores más comunes son:

    • text: Campo de texto simple.
    • email: Puede ingresar más de uno con el atributo "multiple". También puede trabajar con el atributo "pattern".
    • password: Lo que el usuario escribe aparece cifrado (con asteriscos o circulitos).
    • submit: Botón para enviar el formulario.
    • reset: Botón para reiniciar el formulario.
    • checkbox: Casilla de verificación.
    • radio: Botón de opción (radio button).
    • file: Campo para adjuntar archivos.

      • Elemento: input

      Es el elemento más común para crear campos de formulario. Se utiliza junto con el atributo "type" para especificar el tipo de campo.


      No es necesario aclarar el valor de su type ya que por defecto se trata de un valor de tipo texto (enlod emás si es necesario aclarar).

      Atributo: placeholder

      Especifica un texto que se muestra en el campo antes de que el usuario ingrese información. Es útil para dar ejemplos o instrucciones.

      Atributo: value

      Es un valor predeterminado del campo. Es decir, completa el campo con caracteres válidos antes de que lo haga el usuario.

      Atributo: required

      Indica que el campo es obligatorio y debe ser completado antes de enviar el formulario.

      Atributo: pattern

      Entiendo que solo se usa para la password. Permit decidir qué tipo de caracteres van a ser aplicados (eso gd que hacen las páginas de decirte cómo tiene que ser tu contraseña). Para usarlo, hay que ingresar a la página: https://html5pattern.com/ , copiar el patrón que nos guste y pegarlo en el atributo "pattern" del input de tipo password.
